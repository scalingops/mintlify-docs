The Compliance module helps you manage regulatory compliance, track security frameworks, and maintain audit readiness. Organize compliance checks by company, assessment runs, and locations with detailed tracking and evidence management.

## Accessing Compliance

1. Click **Compliance** in the sidebar
2. The compliance dashboard displays your assessment overview

<Frame>
  <img src="/images/compliance/compliance-overview.png" alt="Compliance Dashboard Overview" />
</Frame>

## Dashboard Overview

### Summary Cards

The top section displays four key metrics:

| Card | Description |
|------|-------------|
| **Compliant** | Checks that meet requirements (green) |
| **Not Compliant** | Checks that need attention (red) |
| **In Progress** | Checks currently being assessed |
| **Past Due** | Overdue assessments requiring action |

### Filtering Options

Use the filter bar to narrow your view:

| Filter | Description |
|--------|-------------|
| **Search** | Find specific companies or checks |
| **Company** | Filter by specific company |
| **Framework** | Filter by compliance framework |
| **Status** | Filter by compliance status |
| **Date Range** | Filter by assessment date |

## Compliance Tree Structure

The compliance module uses a hierarchical tree structure to organize assessments:

### Level 1: Companies

Companies appear as the top level of the tree. Click the arrow to expand and see associated compliance runs.

<Frame>
  <img src="/images/compliance/compliance-company-expanded.png" alt="Company with Runs Expanded" />
</Frame>

### Level 2: Compliance Runs

Each company can have multiple compliance runs representing different assessments or time periods:

| Column | Description |
|--------|-------------|
| **Run Name** | Name of the compliance assessment |
| **Total Checks** | Number of checks in the run |
| **Status Badge** | Visual indicator showing compliant/non-compliant counts |

### Level 3: Locations

Runs contain locations representing different areas, departments, or sites being assessed.

### Level 4: Individual Checks

The deepest level shows individual compliance checks with their status:

<Frame>
  <img src="/images/compliance/compliance-tree-expanded.png" alt="Full Compliance Tree Expanded" />
</Frame>

| Column | Description |
|--------|-------------|
| **Check Title** | Name of the compliance requirement |
| **Status** | Compliant, Not Compliant, In Progress, or Past Due |
| **Priority** | Critical, High, Medium, or Low |
| **Due Date** | When the check must be completed |

## Check Details

Click any check to open the detailed view:

<Frame>
  <img src="/images/compliance/compliance-check-detail.png" alt="Compliance Check Detail Dialog" />
</Frame>

### Breadcrumb Navigation

The top of the dialog shows the full path to the check:
- Company > Run > Location Group > Location > Check

### Left Panel Information

| Field | Description |
|-------|-------------|
| **Criticality** | How critical this check is to compliance |
| **Frequency** | How often this check should be assessed |
| **Score Weight** | Point value for compliance scoring |
| **Description** | What this check validates |
| **Why It's Important** | Business and security impact |

### Status and Priority

| Field | Description |
|-------|-------------|
| **Status** | Dropdown to set Compliant, Not Compliant, In Progress |
| **Priority** | Critical, High, Medium, or Low |
| **Due Date** | Target completion date with date picker |

### Public Notes

The Public Notes section supports rich content including AI-generated assessments:

- **Summary** - Overview of the compliance requirement
- **Business Impact** - Effect on business operations if non-compliant
- **Remediation** - Steps to achieve compliance
- **Validation Rules** - How compliance is verified

<Tip>
Use the AI features to automatically generate detailed compliance documentation based on the check requirements and your organization's context.
</Tip>

### Private Notes

Internal notes visible only to your team. Use this for:
- Internal discussions
- Technical implementation details
- Notes not intended for audit reports

### Additional Tabs

| Tab | Description |
|-----|-------------|
| **Uploads** | Attach evidence files and supporting documentation |
| **Custom Fields** | Additional fields configured for your organization |
| **History** | Audit trail of all changes to this check |

## Compliance Scoring

### Score Calculation

Each check has a score weight that contributes to overall compliance:

- **Score Weight** - Points for this check (e.g., 8.00)
- **Earned Points** - Points earned when compliant
- **Total Score** - Sum of all weighted checks

### Severity Impact

Higher severity checks typically have greater score weights:

| Priority | Typical Weight Range |
|----------|---------------------|
| Critical | 8.0 - 10.0 |
| High | 5.0 - 8.0 |
| Medium | 2.0 - 5.0 |
| Low | 0.5 - 2.0 |

## Working with Compliance Runs

### Creating a New Run

1. Click **New Run** in the toolbar
2. Select the company and framework
3. Configure run settings and schedule
4. Add locations to assess
5. Save to create the run

### Managing Locations

- **Add Location** - Include new areas in the assessment
- **Edit Location** - Modify location details
- **Remove Location** - Exclude from current run

## Best Practices

### Assessment Strategy

1. **Start with Critical** - Address critical checks first
2. **Regular Reviews** - Schedule recurring assessments
3. **Document Everything** - Attach evidence for all checks
4. **Track Progress** - Monitor status badges for quick overview

### Evidence Management

- **Organize by Check** - Keep evidence linked to specific requirements
- **Date Your Evidence** - Include timestamps in documentation
- **Version Control** - Maintain history of changes
- **Secure Storage** - Evidence is encrypted at rest

### Remediation Workflow

1. Identify non-compliant checks
2. Review remediation guidance
3. Create action plan with due dates
4. Implement changes
5. Update status and attach evidence
6. Document lessons learned

## Troubleshooting

### Checks Not Appearing

- Verify the run is active
- Check location assignments
- Confirm framework is enabled
- Refresh the page

### Status Not Updating

- Ensure you have edit permissions
- Check for validation errors
- Verify all required fields are complete
- Save changes before closing

### Evidence Upload Issues

- Check file size limits
- Verify supported file types
- Ensure adequate storage quota
- Try alternative file format
